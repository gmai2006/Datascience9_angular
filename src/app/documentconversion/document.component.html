Conversion of Legacy MS Word /PDF to XML

Our firm has developed an open source tool, which automatically converts legacy MS Word, PDF documents to XML. The tool can also generate PDF from generated XML if applicable

How the Conversion Works

As shown in Figure 1, the word conversion tool is a multi-phase process. The first phase called analysis will extract the metadata from the document, It then compares the extracted information against the known patterns. If a match is found, the documents with known patterns will be grouped into buckets and will be forwarded to the next phrase called parsing. If the pattern is not found, the analysis will collect the document and push it to a pattern collector, which will be reviewed by our engineers, who will then extend the pattern recognizer to include the new patterns. The new patterns will be added to the software once the software is updated. Meanwhile, all the known patterns are being processed by the parsing phrase.

In the parsing phrase, each bucket will be parsed by different parsers. The parser will parse the MS Word documents and produce intermediate representation (IR) models. These IR models will be fed into the XML transformer phrase. Documents with errors will be collected by error collector for manual review. In addition to the content of the documents are being stored in IR models, the original document styles will be saved as Cascading style sheet (CSS) in a separate file. The XML transformers will load each XML templates from external source and combine with IR models to produce different XML documents. By separating the IR models from the actual XML output, the tool provides a flexibility to add more XML templates and can produce many XML outputs from one IR model. This approach is called separation of concerns proposed by Dr. Parr (2006).
